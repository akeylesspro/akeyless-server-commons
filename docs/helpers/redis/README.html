src/helpers/redis

Contents

  • index.ts - Barrel export for Redis helper modules
  • initialize.ts - Redis client initialization with dual-client architecture (commander for read/write, listener for Pub/Sub), connection management, retry logic, and automatic subscription setup
  • keys.ts - Redis key and pattern utilities for building consistent key names, collection patterns, update channels, and key scanning operations using SCAN command
  • snapshot.ts - Redis Pub/Sub snapshot integration with automatic fallback to Firebase snapshots. Provides same cache parsing interface as Firebase snapshots for seamless data source switching

Context

Redis helpers allow cache hydration and live updates using Redis Pub/Sub while preserving compatibility with Firebase snapshot flows. The Redis integration provides:

  • Dual-Client Architecture - Separate clients for commands and subscriptions
  • Automatic Fallback - Falls back to Firebase when Redis unavailable
  • Real-time Updates - Pub/Sub for live cache updates
  • Consistent Interface - Same parsing interface as Firebase snapshots
