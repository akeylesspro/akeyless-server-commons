src/helpers/redis/snapshot.ts

Purpose

Consume Redis Pub/Sub updates and hydrate cache using Redis data, with Firebase snapshot fallback when Redis is unavailable or unsupported.

Dependencies

  • Redis helpers and connection flags
  • Firebase snapshot helpers and parsers
  • get_collection_keys, scan_redis_keys
  • logger, Firestore Timestamp

Exports and behavior

  • redis_snapshots_bulk: validates configs, hydrates cache, subscribes to updates.

Internal flow

  • validate_config: ensures Redis readiness and settings allow Redis.
  • parse_redis_snapshot: initial load or incremental update handling.
  • default_parsers: routes to object/array parsers.
  • get_collection_data: SCAN and MGET data from Redis.
  • convert_object_timestamps: converts timestamp-like objects.

Context

Allows Redis to be the live data source while keeping the same cache parsing behavior as Firebase snapshots.
